## Idea 1: "Terminal Modernism"

**Design Movement:** Neo-Brutalism meets Command-Line Aesthetics

**Core Principles:**
- Raw, monospaced typography as a primary design element
- High-contrast dark background with neon accent pops
- Grid-breaking asymmetric layouts with deliberate "unfinished" edges
- Data-forward: every section feels like structured output

**Color Philosophy:**
- Background: Deep charcoal `#0D0F14` — evokes a terminal screen at night
- Primary accent: Electric cyan `#00E5FF` — the classic cursor blink
- Secondary accent: Acid green `#39FF14` — for tags and highlights
- Text: Off-white `#E8EDF2` with muted gray `#6B7280` for secondary content
- Emotional intent: Confidence, technical authority, raw intelligence

**Layout Paradigm:**
- Left-anchored sidebar navigation (always visible on desktop)
- Content area uses a "log feed" metaphor — sections scroll like terminal output
- Hero section: full-width with a blinking cursor animation and typed-text effect
- Projects displayed as "file cards" with monospace metadata

**Signature Elements:**
- Blinking cursor `_` after headings
- Code-block styled skill badges with syntax highlighting
- Section dividers that look like terminal prompt lines `$ >`

**Interaction Philosophy:**
- Hover states reveal "command output" — extra info appears as if typed
- Scroll triggers "load" animations that mimic data streaming in
- Click interactions produce subtle terminal "beep" visual feedback

**Animation:**
- Typewriter effect on hero headline
- Cards slide in from left as if being printed
- Skill bars fill like progress indicators in a CLI tool

**Typography System:**
- Display: `JetBrains Mono` — bold weight for headings
- Body: `JetBrains Mono` — regular weight, tight line-height
- Accent labels: uppercase, letter-spaced, monospace

## Idea 2: "Academic Precision" ← SELECTED

**Design Movement:** Swiss International Typographic Style meets Contemporary Academic Publishing

**Core Principles:**
- Strict typographic hierarchy as the primary organizational tool
- Generous whitespace that signals intellectual rigor and clarity
- Asymmetric grid with strong left-rail anchoring
- Subtle depth through layered paper-like surfaces, not gradients

**Color Philosophy:**
- Background: Warm off-white `#F7F5F0` — like aged academic paper
- Primary: Deep navy `#1A2744` — authority, trust, scholarship
- Accent: Burnt sienna `#C4622D` — warmth, distinction, passion
- Surface: Pure white `#FFFFFF` cards on the warm background
- Muted: `#8A8680` for metadata and secondary text
- Emotional intent: Scholarly credibility, quiet confidence, intellectual warmth

**Layout Paradigm:**
- Horizontal split hero: left side large typographic statement, right side portrait/visual
- Sections use a "publication layout" — wide left margin for labels, content flows right
- Navigation: minimal top bar, anchored left with section labels as running headers
- Projects in a masonry-inspired grid with varying card heights

**Signature Elements:**
- Thick left-border accent lines on section headings (sienna color)
- Small superscript numbering on project cards (like academic citations)
- Ruled lines separating content blocks (thin, elegant)

**Interaction Philosophy:**
- Hover reveals a subtle paper-lift shadow on cards
- Navigation highlights the active section with a sienna underline
- Smooth scroll with section anchors

**Animation:**
- Fade-up entrance for content blocks on scroll
- Staggered card reveals in project grid
- Subtle parallax on hero section

**Typography System:**
- Display: `Playfair Display` — serif, editorial authority for headings
- Body: `Source Sans 3` — clean, readable, academic
- Monospace: `JetBrains Mono` — for code snippets and tech labels
- Scale: Large display (5xl+), clear H2/H3 hierarchy, comfortable body (base/lg)

## Idea 3: "Digital Blueprint"

**Design Movement:** Technical Drawing / Engineering Blueprint Aesthetic

**Core Principles:**
- Blueprint grid as a subtle background texture
- Clean, geometric sans-serif typography with precise spacing
- Blue-dominant palette evoking technical schematics
- Content organized as "specification sheets"

**Color Philosophy:**
- Background: Deep blueprint blue `#0A1628`
- Grid lines: Slightly lighter `#0F1E38` for the blueprint grid texture
- Primary text: Blueprint white `#E8F4FD`
- Accent: Bright blueprint line `#4FC3F7`
- Highlight: Golden annotation `#FFD54F`
- Emotional intent: Precision, technical mastery, systematic thinking

**Layout Paradigm:**
- Full-width sections with blueprint grid overlay
- Content blocks styled as "spec panels" with corner brackets
- Skills displayed as technical specification tables
- Projects shown as "system diagrams" with connecting lines

**Signature Elements:**
- Corner bracket decorations on cards `⌐ ¬`
- Dotted annotation lines pointing to key information
- Coordinate-style labels in corners of sections

**Interaction Philosophy:**
- Hover activates "highlight" mode — element glows like a selected schematic component
- Scroll reveals sections with a "drawing" animation
- Active states use a crosshair cursor

**Animation:**
- SVG line-drawing animations for section transitions
- Elements "assemble" on scroll as if being drafted
- Blueprint grid fades in on load

**Typography System:**
- Display: `Rajdhani` — geometric, technical, bold
- Body: `IBM Plex Sans` — engineered clarity
- Labels: `IBM Plex Mono` — technical annotations
- All caps for section labels with wide letter-spacing
